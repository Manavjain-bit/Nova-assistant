# Nova — AI Voice Personal Assistant

Build status: **All 8 phases complete.** See `implementation_plan.md` for the original roadmap this was built against.

## What's implemented so far

### Phase 1 — Base Project Setup, Database Schema, & Security
- FastAPI backend skeleton (`backend/app`), Dockerfiles, docker-compose orchestration (Postgres + Redis + backend + frontend).
- Full SQLAlchemy schema for **all 12 tables** from the plan: `users, tasks, habits, habit_logs, goals, milestones, reminders, notes, conversations, messages, memory_entries, notifications`.
- Alembic configured and an initial autogenerated migration is included (`backend/database/migrations/versions/`); verified to match the models exactly (`alembic check`).
- SQLite is the default `DATABASE_URL` for zero-config local dev/tests; Postgres is used via Docker Compose.
- JWT auth: register/login with bcrypt password hashing (`/auth/register`, `/auth/login`). Google OAuth stub routes (`/auth/google/login`, `/auth/google/callback`) — return a clear "not configured" response until real `GOOGLE_CLIENT_ID/SECRET` are supplied.
- Minimal Next.js (App Router) + TypeScript + Tailwind + Zustand + React Query frontend scaffold (full dashboard UI is Phase 7).

### Phase 2 — Task Manager & AI Smart Prioritization
- Full task CRUD + subtasks: `POST/GET/PATCH/DELETE /tasks`, `GET /tasks/{id}/subtasks`, filtering by status/category, and a `?prioritized=true` sorted view.
- **Smart Prioritization Engine** (`backend/app/services/prioritization.py`): scores every task 0–100 from a weighted blend of urgency (time-to-due-date decay curve), importance (priority label), estimated duration (quick-win boost), and dependency depth (subtask boost). Recomputed on create/update and via the prioritized listing endpoint.

### Phase 3 — Reminder System & Scheduler
- **Natural-language reminder parser** (`backend/app/services/reminder_parser.py`): regex/rule-based, handles relative phrases ("in 2 hours"), vague time anchors ("tomorrow after lunch"), daily/weekly/monthly recurrence ("every day at 9am", "every monday", "on the 1st of every month"), and computes both `next_trigger` and a cron expression for recurring reminders.
- `POST /reminders/parse` creates a reminder directly from free text; `POST /reminders` accepts structured input.
- **APScheduler background job** (`backend/app/services/scheduler.py`) runs every minute, finds due reminders, creates `Notification` rows, and reschedules recurring ones using their cron expression. `POST /reminders/dispatch-due` lets you trigger a sweep on demand (handy for demos/tests instead of waiting on the interval).
- Notification feed: `GET /notifications`, `PATCH /notifications/{id}/read`.

### Phase 4 — Goals, Habits, Notes, & Mock Integrations
- **Habits**: CRUD + `POST /habits/{id}/logs` to log completion/miss days. Streak logic (`backend/app/services/habit_tracker.py`) correctly increments on consecutive days, resets after a gap, and is idempotent for same-day re-logging.
- **Goals & milestones**: CRUD for goals, `POST /goals/{id}/milestones`, and `PATCH /goals/{id}/milestones/{id}/complete` which auto-recomputes `progress_percentage` from completed-milestone ratio.
- **Notes**: CRUD with pinning, tag filtering, and substring search across title/content (`GET /notes?q=...&tag=...&pinned_only=...`).
- **Gmail & Calendar connectors** (`backend/app/services/gmail.py`, `calendar.py`): structured service classes that serve realistic simulated data by default; designed to be swapped for the real Google APIs once `GOOGLE_CLIENT_ID/SECRET` + a stored refresh token are present — the route contracts (`/integrations/gmail/messages`, `/integrations/calendar/events`) won't need to change.
- **File Search** (`backend/app/services/file_search.py`): scans `FILE_SEARCH_WORKSPACE_DIR`, filters to PDF/image/TXT, indexes filenames and (for .txt) content, and supports substring search via `GET /integrations/files/search?q=...`.

**33 passing tests** across auth, tasks, prioritization, reminder parsing, reminders API + scheduler dispatch, habits/streaks, goals/milestones, notes, and integrations (`backend/app/tests`).

### Phase 5 — AI Engine, Memory Manager, & Daily Briefing / Night Review
- **AI core service** (`backend/app/services/ai.py`): wraps OpenAI chat completions when `OPENAI_API_KEY` is set; otherwise falls back to a rule-based intent matcher (reminder/task/habit/goal/note/greeting/smalltalk) so chat still responds usefully offline. Network/auth errors degrade gracefully rather than failing the request.
- **Memory Manager** (`backend/app/services/memory.py`):
  - *Short-term*: pulls the last N turns of a conversation as chat-style context.
  - *Long-term*: regex-based fact extraction (name, work, occupation, friends, preferences/dislikes) upserted into `memory_entries`.
  - *Summarization*: once a conversation passes 30 messages, older messages are compressed into `conversations.summary` and deleted, keeping the most recent 10 verbatim.
- **Chat API** (`/chat/message`, `/chat/conversations`, `/chat/conversations/{id}/messages`, `/chat/memory`): persists every turn, runs memory extraction + summarization automatically, and returns which source (`openai` / `local_fallback` / `local_fallback_error`) produced the reply.
- **Daily Briefing** (`GET /briefing/morning`, `backend/app/services/briefing.py`): combines today's calendar events (mock or live), the top 5 prioritized tasks, reminders due in the next 24h, and a plain-English workload summary, with a time-of-day-aware greeting.
- **Night Review** (`POST /briefing/night-review`): accepts mood + productivity rating, reports tasks completed today vs. still pending/overdue, and offers a rollover suggestion.

### Phase 6 — Voice Assistant (STT / TTS)
- **Backend voice service** (`backend/app/services/voice.py`, router at `/voice/*`): `POST /voice/stt` wraps OpenAI Whisper for transcription; `POST /voice/tts` wraps OpenAI TTS for speech synthesis. Both return a structured fallback response (`source: "fallback"` + a `fallback_reason`) instructing the client to use the browser's Web Speech API when no key is configured — matching the plan's graceful-degradation requirement. `GET /voice/capabilities` reports what's available server-side.
- **Frontend voice hook** (`frontend/hooks/useVoiceAssistant.ts`): wraps `SpeechRecognition` and `SpeechSynthesis` directly in the browser (the always-available path, zero backend keys required), and implements **"Hey Nova" wake-word detection** via a continuous-listening recognizer that watches interim transcripts for the wake phrase before triggering the main listen-and-respond flow.
- **VoiceButton component** (`frontend/components/VoiceButton.tsx`): functional glowing voice button wired end-to-end — tap to talk, transcript goes to `/chat/message`, reply is spoken back via the synthesis API. Visual polish (continuous voice waves, Framer Motion) lands in Phase 7's dashboard pass.

**44 passing tests** total — the above plus memory fact-extraction, chat API (fallback replies, conversation continuity, long-term memory persistence), daily briefing/night-review content, and voice STT/TTS fallback behavior (`backend/app/tests`).

## Running it

### Local (no Docker)
```bash
cd backend
pip install -r requirements.txt --break-system-packages   # or use a venv
uvicorn app.main:app --reload
# -> http://localhost:8000/health, interactive docs at /docs
```
SQLite (`nova.db`) is used automatically — no setup required.

### Tests
```bash
cd backend
pytest app/tests -v
```

### Docker Compose (Postgres + Redis + backend + frontend)
```bash
cp .env.example .env   # fill in OPENAI_API_KEY etc. only if you have them — fallback mode works without
docker compose -f docker/docker-compose.yml up --build
# backend -> http://localhost:8000
# frontend -> http://localhost:3000
```

### Migrations
```bash
cd backend
alembic upgrade head                 # apply
alembic revision --autogenerate -m "message"   # create a new one after model changes
```

### Phase 7 — Next.js Frontend Dashboard UI
- Full premium dashboard at `/dashboard` (plus a landing page and `/login` register-or-sign-in flow): icon sidebar nav, top bar with notifications + dark/light toggle, a morning-briefing hero pulling live data from `/briefing/morning`, a prioritized task list driven by the Phase 2 scoring engine, habit/goal progress rings (animated SVG circles, shared `ProgressRing` component), a notes grid, and **VoiceOrb** — the signature glowing, ambient-pulsing voice button (Framer Motion) docked at the bottom of every dashboard view, wired end-to-end to `/chat/message` and the Phase 6 voice hook.
- Design system: CSS-variable-driven dark/light themes (`app/globals.css` + `tailwind.config.js`), "Space Grotesk" display / "Inter" body / "JetBrains Mono" data type pairing, persisted theme preference via `useThemeStore` + `localStorage`.
- Verified: `npm run build` compiles cleanly (0 errors) and `npm run lint` reports no warnings.

### Phase 8 — Verification & Packaging
- **93% backend statement coverage** (`pytest --cov=app`, gated at a 90% floor via `.coveragerc`'s `fail_under`), **53 passing tests** spanning every router and service, including 404/edge-case paths for tasks, habits, goals, notes, reminders, and chat.
- `scripts/verify.sh`: one-shot script that installs deps, runs backend tests with the coverage gate, then runs frontend lint + production build — exits non-zero on any failure.
- `scripts/seed_db.py`: seeds a demo user (`demo@nova.ai` / `demo12345`) with sample tasks, a habit streak, a goal with milestones, and a note, so the dashboard isn't empty on first run.
- Docker Compose packaging (Postgres + Redis + backend + frontend), `.env.example`, and this README serve as the deployment guide.
